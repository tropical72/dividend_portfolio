## 2. 세무 알고리즘 핵심 이슈 분석: 금융소득 종합과세 오류

### 2.1. 문제 현황 및 원인 분석
[tax_engine.py](file:///home/tropical72/Work/dividend_portfolio/src/core/tax_engine.py)의 [calculate_us_dividend_tax](file:///home/tropical72/Work/dividend_portfolio/src/core/tax_engine.py#L249-L286) 메서드에서 금융소득 종합과세 대상자(연 금융소득 > 2,000만 원)의 산출세액을 계산하는 로직에 결함이 존재합니다.

#### 현재 구현 코드 ([tax_engine.py:L266-L268](file:///home/tropical72/Work/dividend_portfolio/src/core/tax_engine.py#L266-L268))
```python
incremental_tax = self.calculate_progressive_income_tax(base + financial_total) - (
    self.calculate_progressive_income_tax(base)
)
```

#### 대한민국 소득세법 기준 (비교과세 제도)
대한민국 소득세법 제62조(세액계산의 특례)에 따르면, 금융소득 종합과세 시 금융소득 전체를 종합소득 세율로 과세하지 않습니다. **기본적으로 2,000만 원까지는 원천징수세율(14%, 지방소득세 포함 15.4%)을 적용**하고, **2,000만 원을 초과하는 부분만 다른 종합소득과 합산**하여 누진세율을 적용한 뒤, 이를 원천징수세율로만 과세했을 때(비교과세)의 세액과 비교하여 더 큰 금액을 산출세액으로 합니다.

즉, 종합과세 산출세액은 다음과 같이 `Max(일반산출세액, 비교산출세액)`로 계산되어야 합니다.
1. **일반산출세액:** `누진세액(다른 종합소득 + 금융소득 - 2,000만 원) + (2,000만 원 * 14%)`
2. **비교산출세액:** `누진세액(다른 종합소득) + (금융소득 * 14%)`

현재 코드는 `financial_total` 전체를 누진세율 구간에 얹어 계산하므로, **2,000만 원 이하 구간까지 종합소득 한계세율로 과세되어 세액이 과대평가(Overestimate)되는 치명적인 오류**를 범하고 있습니다.

#### 실제 수치 대조 (예: 타 종합소득 5,000만 원, 금융소득 2,400만 원일 때)
- **실제 세법 기준 증분세액:** 지방소득세 10% 포함 기준 **4,136,000원**
- **현재 엔진 계산 결과:** 지방소득세 10% 포함 기준 **6,336,000원** (약 53% 수준인 220만 원 과다 청구)

## 5. 조치 결과 (2026-06-22)

- `calculate_us_dividend_tax`를 소득세법 제62조 비교과세 구조에 맞춰 수정했다. 기준금액 이하 금융소득은 국내 배당세율로 분리하고 기준 초과분만 기타 종합소득 과세표준과 누진 합산한다.
- 예시 조건(기타 종합소득 과세표준 5,000만 원, 금융소득 2,400만 원)의 국내 산출세액은 4,136,000원, 미국 원천세 공제 후 국내 추가세액은 536,000원으로 정밀 검증한다.
- 일반산출세액 11,000,000원, 비교산출세액 10,560,000원, 금융소득 증분세액 4,136,000원을 연간 세금 감사 API와 Retirement 감사표에 노출한다.
- 기타 종합소득이 없는 경우 비교산출세액이 선택되는 경계와, 외부 금융소득이 함께 있는 경우 미국 배당 비중 배분도 별도 테스트로 고정했다.
- 전체 Pytest, Ruff, Black, Frontend lint/build/Prettier를 통과했다. 기존 ESLint 경고 2건과 번들 크기 경고는 이번 변경 범위 밖으로 유지한다.
