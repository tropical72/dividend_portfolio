# Test Cases: Personal vs Corporate Cost Comparison Simulator

## 1. 개요 및 테스트 전략
본 문서는 `개인운용 100% vs 법인운용 100%` 비교 시뮬레이터의 계산 정합성과 UI 신뢰성을 검증하기 위한 테스트 시나리오를 정의한다.

테스트의 핵심 목적은 다음과 같다.

- 동일 기준 포트폴리오와 동일 `TR`이 양 시나리오에 공정하게 적용되는지 검증
- 개인/법인 차이가 투자 성과가 아니라 세금, 건강보험료, 급여, 운영비 구조에서만 발생하는지 검증
- 가구 단위 KPI와 누적 비교 결과가 일관되게 렌더링되는지 검증
- 저장/재로드/테스트 격리를 통해 사용자 실데이터가 오염되지 않는지 검증

---

## 2. 도메인별 테스트 시나리오

### [Structure 1] 입력 모델 및 저장 정합성 (REQ-CCS-10 ~ 14, 70 ~ 72)
- **[TEST-CCS-01] 비교 설정 저장/재로드 정합성:**
    - 비교 시뮬레이터 설정 저장 후 재조회 시 동일 값이 유지되는지 확인.
    - 가구 구성원, 부동산, PA, 시뮬레이션 기간, 급여 수령자 목록이 누락 없이 복원되는지 확인.
- **[TEST-CCS-02] 저장 구조 분리:**
    - 비교 시뮬레이터 설정 저장이 `retirement_config.json` 또는 기존 `settings.json` 핵심 필드를 오염시키지 않는지 확인.
- **[TEST-CCS-03] 기본값/빈 상태 검증:**
    - 새 탭 최초 진입 시 더미 가구 구성원, 더미 급여 수령자, 더미 부동산 값이 자동 주입되지 않는지 확인.
    - 기본값이 존재하는 경우에도 그 값이 문서화된 정책과 일치하는지 확인.
- **[TEST-CCS-04] 테스트 상태 복구:**
    - Playwright 또는 API 테스트가 비교 시뮬레이터 설정을 변경해도 종료 후 사용자 데이터가 복구되는지 확인.

### [Structure 2] 기준 포트폴리오 및 수익률 모델 (REQ-CCS-20 ~ 24, 91 ~ 92)
- **[TEST-CCS-10] 활성 master portfolio 연동:**
    - 비교 시뮬레이터가 현재 활성 master portfolio를 기준 포트폴리오로 사용하고 있는지 확인.
- **[TEST-CCS-11] DY 계산 정합성:**
    - 지정된 종목 비중과 배당수익률을 가진 포트폴리오에서 계산된 `DY`가 수학적으로 일치하는지 확인.
- **[TEST-CCS-12] TR 계산 정합성:**
    - `TR = DY + PA` 규칙이 정확히 적용되는지 확인.
- **[TEST-CCS-13] 동일 TR 강제 검증:**
    - 동일 포트폴리오/동일 PA 입력 시 개인운용과 법인운용이 같은 `TR`을 사용하고 있는지 확인.
- **[TEST-CCS-14] 기준 포트폴리오 누락 에러:**
    - 활성 master portfolio가 없거나 참조가 깨졌을 때 비교 실행이 차단되고 명시적 오류가 반환되는지 확인.

### [Structure 3] 개인운용 시나리오 (REQ-CCS-30 ~ 34, 90)
- **[TEST-CCS-20] 개인 총수익 계산:**
    - 개인 투자자산 총액과 기준 `TR`을 이용한 연간 총수익 계산이 기대값과 일치하는지 확인.
- **[TEST-CCS-21] 지역건강보험료 반영:**
    - 개인운용 시나리오에서 부동산 공시지가와 지분율이 건보료 결과에 반영되는지 확인.
- **[TEST-CCS-22] 가구 기준 결과 집계:**
    - 개인운용 결과가 본인 단독이 아니라 가구 전체 관점 KPI로 집계되는지 확인.
- **[TEST-CCS-23] 개인연금 분리 처리:**
    - 개인연금 자산 평가액이 비교 운용자산 또는 부동산 재산 점수에 잘못 포함되지 않는지 확인.
- **[TEST-CCS-24] 세후 순증가액 산출:**
    - 개인운용에서 세금/건보료 차감 후 `세후 순증가액`이 일관되게 계산되는지 확인.

### [Structure 4] 법인운용 시나리오 (REQ-CCS-40 ~ 47, 93 ~ 94)
- **[TEST-CCS-30] 법인 총수익 계산:**
    - 법인 투자자산 총액과 기준 `TR`을 이용한 연간 총수익 계산이 기대값과 일치하는지 확인.
- **[TEST-CCS-31] 급여 수령자 수 제한:**
    - 법인 급여 수령자를 최대 4명까지 허용하고, 5명 이상 입력은 차단되는지 확인.
- **[TEST-CCS-32] 직장건강보험료 합산 규칙:**
    - 급여 수령자별 직장건강보험료가 회사 부담분과 개인 부담분으로 분리 계산되고 총가구 비용에 모두 반영되는지 확인.
- **[TEST-CCS-33] 급여 수령자 변화 반영:**
    - 급여 수령자를 1명, 2명, 4명으로 바꿨을 때 총비용과 실수령액이 규칙대로 변경되는지 확인.
- **[TEST-CCS-34] 법인 운영비 및 법인세 반영:**
    - 월 고정 운영비와 법인세가 연 총비용에 반영되는지 확인.
- **[TEST-CCS-35] 주주대여금 상환 제한:**
    - 주주대여금 상환이 초기 대여금 잔액과 법인 현금흐름 범위를 초과하지 않는지 확인.
- **[TEST-CCS-36] 유보이익과 실수령 분리:**
    - 법인 내부 유보이익이 월 실수령액에는 포함되지 않고 `세후 순증가액` 또는 누적 순자산에는 반영되는지 확인.
- **[TEST-CCS-37] 급여 수령자 0명 경고:**
    - 법인운용 시나리오에서 급여 수령자가 없을 경우 계산은 가능하되 경고 또는 주의 문구가 노출되는지 확인.

### [Structure 5] 비교 엔진 및 누적 시뮬레이션 (REQ-CCS-50 ~ 55)
- **[TEST-CCS-40] KPI 비교 카드 정합성:**
    - `월 실수령액`, `연 총비용`, `건강보험료`, `세후 순증가액`이 양 시나리오 카드에 모두 표시되는지 확인.
- **[TEST-CCS-41] 연간 우위 판정:**
    - 입력 조건에 따라 어느 시나리오가 연간 기준으로 유리한지 올바르게 판정되는지 확인.
- **[TEST-CCS-42] 누적 우위 판정:**
    - 시뮬레이션 기간을 늘렸을 때 누적 기준 판정 결과가 함께 계산되는지 확인.
- **[TEST-CCS-43] 차이 항목 분해:**
    - `건보료`, `세금`, `총급여`, `회사부담 보험`, `운영비` 분해값이 응답에 포함되는지 확인.
- **[TEST-CCS-44] 상위 차이 요인 설명:**
    - 상위 차이 유발 요인 3개가 explanation payload 및 UI 요약 패널에 노출되는지 확인.
- **[TEST-CCS-45] 기간 변경 반영:**
    - 시뮬레이션 기간을 변경하면 누적 비용, 누적 순자산, 누적 차이 그래프가 함께 갱신되는지 확인.

### [Structure 8] 비교 모드 및 상세 감사 (REQ-CCS-4.1, 55, 95)
- **[TEST-CCS-90] 비교 모드 전환(Target vs Asset) 테스트:**
    - 상단 스위처를 통해 모드를 전환할 때 입력 필드(목표 수익 vs 자산 총액)의 활성/비활성 및 결과 갱신 확인.
- **[TEST-CCS-91] 상세 비용 감사(Audit) 패널 검증:**
    - KPI 카드 하단 또는 상세 섹션에 건보료 점수 산출 내역(재산/소득/자동차 등)과 점수당 단가가 정확히 표시되는지 확인.
- **[TEST-CCS-92] Asset-driven 순방향 계산 정합성:**
    - 동일 자산과 TR 조건에서 총수익 대비 세후 실수령액 계산 결과가 수동 검산값과 일치하는지 확인.
- **[TEST-CCS-93] 30년 장기 프로젝션 정합성:**
    - 복리 효과가 반영된 30년 자산 추이 데이터가 논리적으로 타당하게 생성되고 그래프에 반영되는지 확인.
- **[TEST-CCS-94] Asset-driven 모드 wiring 및 KPI 단위 회귀 테스트:**
    - UI에서 Asset-driven을 저장/실행했을 때 백엔드 응답의 `simulation_mode`가 `asset`인지 확인.
    - Asset-driven KPI 카드의 첫 번째 값이 `월 실수령액` 기준으로 표시되고, 연간 값으로 잘못 12배 표기되지 않는지 확인.
- **[TEST-CCS-95] 개인 Asset-driven 시계열 이중 계산 회귀 테스트:**
    - 개인 Asset-driven 장기 시계열에서 동일한 세후 현금이 누적 가계현금과 자산 증가에 동시에 반영되지 않는지 확인.
    - `total_economic_value`가 같은 현금을 이중 합산하지 않는지 확인.
- **[TEST-CCS-96] 승자 요약 명시성 회귀 테스트:**
    - 결과 요약 영역이 `누가 우세한지`만이 아니라 `연 순현금흐름 기준 차액`과 `누적 순현금흐름 기준 차액`을 함께 명시하는지 확인.
    - 사용자가 툴팁을 열지 않아도 승자, 비교 기준, 우위 금액을 바로 읽을 수 있는지 확인.
- **[TEST-CCS-97] 순현금흐름 기준 우열 판정 회귀 테스트:**
    - 우열 판정의 기준이 `annual_total_cost`가 아니라 `annual_net_cashflow`인지 확인.
    - `주주대여금 상환`이 top driver에서 제외되고, 법인 메인 비교값은 `net_corporate_cash + net_salary`와 일치하는지 확인.
- **[TEST-CCS-98] 공통 비교 철학 회귀 테스트:**
    - `목표 수익 기반`과 `보유 자산 기반` 모두에서 메인 비교 기준이 동일한 순현금 철학을 사용하는지 확인.
    - 메인 비교 UI에 `연간 주주대여금 상환` 입력/카드가 노출되지 않는지 확인.

### [Structure 9] UI/UX 및 시각화 (REQ-CCS-60 ~ 67)
- **[TEST-CCS-50] 독립 탭 렌더링:**
    - 전역 네비게이션에 비교 시뮬레이터 탭이 노출되고 정상 진입 가능한지 확인.
- **[TEST-CCS-51] 입력 섹션 구조 검증:**
    - `가구 정보`, `개인 자산`, `부동산`, `기준 포트폴리오 / 수익률`, `법인 설정`, `시뮬레이션 기간` 섹션이 구조적으로 렌더링되는지 확인.
- **[TEST-CCS-52] assumptions 배지 검증:**
    - 결과 화면에 `기준 포트폴리오 이름`, `DY`, `PA`, `TR`, `시뮬레이션 기간`, `기준 연도`가 표시되는지 확인.
- **[TEST-CCS-53] 비용 분해 차트 렌더링:**
    - 비용 분해 차트가 양 시나리오의 비용 구조를 시각적으로 비교할 수 있게 렌더링되는지 확인.
- **[TEST-CCS-54] 워터폴 차트 렌더링:**
    - 총수익에서 비용 차감 후 실수령으로 이어지는 워터폴 차트가 올바르게 렌더링되는지 확인.
- **[TEST-CCS-55] 누적 비교 그래프 렌더링:**
    - 기간별 누적 순자산 또는 누적 실수령 비교 그래프가 렌더링되는지 확인.
- **[TEST-CCS-56] 에러/빈 상태 UX:**
    - 기준 포트폴리오 누락, 필수값 부족, 음수 입력 등의 오류 상태에서 원인과 해결 방법이 표시되는지 확인.

### [Structure 7] 정책/메타/컴플라이언스 (REQ-CCS-80 ~ 82)
- **[TEST-CCS-60] 기준 연도 노출:**
    - 건강보험 및 세율 계산의 기준 연도가 결과 UI 또는 메타에 표시되는지 확인.
- **[TEST-CCS-61] 정책 수치 출처 노출:**
    - 건강보험료율, 장기요양보험 비율, 세율 등의 출처가 도움말 또는 메타에 표시되는지 확인.
- **[TEST-CCS-62] 추정치 경고:**
    - 결과가 확정 세무 자문이 아닌 시뮬레이션 추정치임을 알리는 경고가 사용자에게 노출되는지 확인.

## 3. 검증 도구 및 환경
- **Pytest (Backend):** 비교 엔진 계산 로직, 저장 스키마, API 정합성 검증.
- **Playwright (Frontend):** 독립 탭 UI, 저장/재로드, KPI 카드, 차트 렌더링, 오류 UX 검증.
- **상태 격리:** 테스트는 snapshot/restore 또는 테스트 전용 설정 파일을 사용하여 사용자 실데이터와 분리되어야 한다.

## 4. Feature Completion Gate
- 비교 시뮬레이터 관련 Python 변경은 커밋 전 `PYTHONPATH=. .venv/bin/ruff check <changed_python_files>`와 `PYTHONPATH=. .venv/bin/black --check <changed_python_files>`를 통과해야 한다.
- 비교 시뮬레이터 관련 프런트엔드 변경은 `npm run lint`, `npm run build`, `npx prettier --check <changed_frontend_files>`를 통과해야 한다.
- 기능 완료 증명은 최소 1개의 관련 `pytest` 묶음과 1개의 관련 `Playwright` 시나리오 통과로 마감한다.
